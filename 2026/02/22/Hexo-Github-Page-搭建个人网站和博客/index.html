很早之前，就想过要搭建一个个人网站和博客了，但是网上的教程大部分都是推荐购买云服务器的，这对于一个普通学生来说实在是没有什么性价比，所以一直未能付诸行动。最近才偶然看到 github pages 提供免费的域名，非常心动，于是说干就干，开始查找资料，着手搭建自己的个人网站和博客。在这篇文章中，我记录了自己如何从零开始使用 Hexo 框架配合 GitHub Pages 搭建一个免费个人博客的过程。

1. 为什么选择 Hexo + GitHub Pages？

  • 免费：不需要购买服务器和域名（GitHub Pages 提供免费二级域名）。
  • 静态：Hexo 生成的是静态网页，加载速度快，安全性高。
  • 可定制：拥有丰富的主题（如本站使用的 Redefine）和插件生态。
  • Markdown：专注于写作，使用 Markdown 语法即可撰写文章。

9.2 浏览器图标（Favicon）不更新

如果你更换了 favicon.ico 但浏览器显示的还是旧图标，通常是因为浏览器缓存。

  1. 确保图片路径配置正确（如 _config.redefine.yml 中的 defaults.favicon）。
  2. 运行 hexo clean && hexo g 重新生成。
  3. 使用 Ctrl + F5 强制刷新浏览器。

9.3 导航栏出现两个“首页”

如果在配置 Redefine 主题时导航栏出现了两个首页，可能是因为主题默认配置与你的自定义配置合并了。
解决方法：在 node_modules/hexo-theme-redefine/_config.yml 中将 navbar.links 设为空，或者确保只在根目录的 _config.redefine.yml 中定义导航。
